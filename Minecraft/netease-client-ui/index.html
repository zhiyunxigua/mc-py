作者：西瓜

开始

UI主要分为两种分类，一个是普通的，另一个是堆栈形式。

普通的UI就像玩家的物品栏/血条/计分板，这些，在玩家的主界面(hud)叠加显示，并且不影响玩家的操作

堆栈的顾名思义就是堆栈形式，新创建的会在最顶层显示，并且隐藏其他UI，自动弹出鼠标指针，类似原版的箱子/设置/背包界面

开始制作

每个UI都应该现有一个初始这样的结构

Example.json 
{
    "namespace": "Example",  // 命名空间，需要保持唯一
    "main@common.base_screen": {  // 主screen，继承原版的base_screen，可以有原版的特性，比如适配异形屏
        "$screen_content": "Example.main_screen"  // 主screen的内容是Example的main_screen控件
    },
    "main_screen": {  // 自定义控件
        "type": "screen",  // 类型
        "controls": []  // 内容
    }
}
